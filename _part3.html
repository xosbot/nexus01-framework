Karpathy's Guidelines — 41% → 11% Error Reduction

The rules that make agents 4x less likely to make mistakes

THINK BEFORE CODING

State assumptions explicitly. Present alternatives. Push back when simpler approach exists.

Before: Claude picks interpretation silently
After: Claude states assumptions, asks when unclear

SIMPLICITY FIRST

No features beyond what was asked. No abstractions for single-use code. No speculative flexibility.

Before: 200 lines with error handling for impossible scenarios
After: 50 lines that solve the actual problem

SURGICAL CHANGES

Touch only what the request requires. Match existing style. Every changed line traces to the request.

Before: Fix validation bug → reformat module, rename variables
After: Fix validation bug → one line changed, bug fixed

GOAL-DRIVEN EXECUTION

Define verifiable success criteria. Let the model loop until criteria are met.

Before: Step 1: create file. Step 2: add function.
After: Goal: Write a test that reproduces the issue, then make it pass.
Error Rate: 41% → 11% — Verified across 40 OpenClaw PRs

Playwright MCP gives agents precise, deterministic control over web pages through the Chrome DevTools Protocol.

Architecture
Agent → Playwright MCP Server → Browser (CDP) → Website

Best for: Web scraping, form filling, precise interaction

Visual Desktop Control
