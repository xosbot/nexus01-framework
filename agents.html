Message Format All inter-agent communication uses a typed Message object with type (task, intel, alert, heartbeat) and payload (structured data). Every message is logged, timestamped, and traceable. The orchestrator routes based on message type and agent availability.

Cold Mode Protocol

The system's immune system against hallucinations. Every decision verified. Every action deliberate.

01
DATA VERIFICATION
Is the source reliable? Cross-validated? The agent checks data provenance, cross-references with known trusted sources, and validates freshness before proceeding.
02
PARAMETER VALIDATION
Within standard deviation ranges? Input parameters are checked against historical baselines. Out-of-range values trigger alerts and require manual override.
03
CONFIDENCE ASSESSMENT
Model confidence ≥ 0.8? The LLM's own confidence score is evaluated. Below threshold? The system defers to deterministic scripts or requests human input.
04
RISK EVALUATION
What are the consequences of wrong action? Risk is scored on a 0-1 scale. High-risk actions require explicit approval and trigger enhanced monitoring.
05
FALLBACK DETERMINATION
Is a deterministic script ready? If Cold Mode fails, the system checks for pre-built fallback scripts that can achieve the same goal without LLM reasoning.

✓ PASS

All five gates cleared. Execute the action. Log the decision trail. Continue monitoring.

✗ FAIL

Block the action. Alert the operator. Attempt fallback. Log the failure reason.

Activation Triggers

🔒 Any action with write/execute permissions
💰 Financial transactions or contract calls
📡 Cross-agent intelligence relay
⚡ High-frequency or batch operations
🌐 External API calls with side effects
🗑️ Any destructive or irreversible action
