The Build Plan

12 weeks from zero to production. Every week has a deliverable.

Week-by-Week Roadmap

Four phases. Twelve weeks. Zero ambiguity. Every week ends with something working.

Phase 1: The Skeleton
Weeks 1–3
WEEK 1

Infrastructure & Bootstrap

  • Provision VPS (Hetzner CX22, €4.50/mo)
  • SSH hardening (ED25519 keys only)
  • Initialize /opt/nexus/ project structure
  • Docker Compose for local dev
  • Deploy CI/CD skeleton (GitHub Actions)
Deliverable VPS accessible, project structure ready
WEEK 2

Core Messaging

  • Extract XClaw core modules (LLM Router, Agent Loop, Tool Registry)
  • Implement Redis pub/sub message bus
  • Agent base class with lifecycle hooks
  • Heartbeat/health check system
Deliverable 2 agents can exchange typed messages via Redis
WEEK 3

Config & Cold Mode

  • Centralized config (pydantic-settings, .env + YAML)
  • Cold Mode protocol implementation
  • Structured logging (structlog → journald)
  • Phase 1 integration tests
Deliverable Cold Mode gates all agent decisions
Phase 2: The Brain
Weeks 4–6
WEEK 4

LLM Integration

  • Unified LLM client with provider abstraction
  • Gemini Pro (primary), Claude 3.5 (reasoning), Perplexity (search)
  • Retry logic with exponential backoff + circuit breaker
  • Token counting and cost tracking
Deliverable LLM routing works with 3 providers
WEEK 5

RAG Infrastructure

  • Supabase project with pgvector extension
  • Document ingestion pipeline (chunking → embedding → storage)
  • Embed text-embedding-3-large (OpenAI)
  • Ingest all tradecraft docs
Deliverable Query "What is Cold Mode?" returns relevant chunks
WEEK 6

Agent Memory

  • Short-term: Redis conversation buffer
  • Long-term: Supabase vector store
  • Working memory: Agent-scoped scratchpad
  • Memory consolidation: periodic summarization
Deliverable Agent memory persists across restarts
Phase 3: The Hands & Eyes
Weeks 7–10
WEEK 7

OSINT Agent

  • Web scraping (httpx + BeautifulSoup4)
  • Social media monitoring
  • Financial intelligence integration
  • Structured intelligence reports
Deliverable Agent gathers and structures intelligence
WEEK 8

Execution Agent + Telegram

  • Shell command execution with permission levels
  • Sandboxed execution environments
  • Telegram bot interface
  • Human-in-the-loop approval
Deliverable Telegram command → agent receives → response
WEEK 9

Analytics Agent

  • Pattern recognition on collected data
  • Anomaly detection (Z-score on rolling windows)
  • Daily summary generation
Deliverable Analytics detects synthetic anomalies
WEEK 10

Protocol Alpha + Integration

  • Timing Engine (See → Sense → Decide → Act)
  • Full Cold Mode wiring across all agents
  • End-to-end integration tests
Deliverable Full pipeline: Telegram → OSINT → Analytics → Execution
Phase 4: Harden & Ship
Weeks 11–12
WEEK 11

Observability & Security

  • Prometheus + Grafana monitoring
  • Security audit (no secrets in code)
  • RBAC implementation
  • Audit trail logging
Deliverable Monitoring dashboard, security hardened
WEEK 12

Documentation & Deploy

  • Operational runbook
  • Automated backup strategy
  • Deployment script (zero-downtime)
  • Final integration test suite
Deliverable Production-ready framework

Phase Gate Criteria

Every phase ends with a gate. Pass all criteria or don't ship.

Phase 1: The Skeleton

Weeks 1–3
  • ● VPS accessible via SSH (ED25519)
  • ● Redis pub/sub working between agents
  • ● Cold Mode gates all agent decisions
  • ● Integration tests passing
  • ● Docker Compose starts all services

Phase 2: The Brain

Weeks 4–6
  • ● LLM routing works with 3 providers
  • ● Documents ingested and queryable via RAG
  • ● Agent memory persists across restarts
  • ● Token costs tracked per request
  • ● Structured logging to journald

Phase 3: Hands & Eyes

Weeks 7–10
  • ● OSINT Agent gathers and structures intel
  • ● Telegram → agent → response working
  • ● Analytics detects synthetic anomalies
  • ● Full pipeline: Telegram → OSINT → Analytics → Execution
  • ● Human-in-the-loop approval for high-risk actions

Phase 4: Harden & Ship

Weeks 11–12
  • ● Prometheus + Grafana monitoring active
  • ● Security audit complete (no secrets in code)
  • ● Automated backups configured and tested
  • ● Zero-downtime deployment script working
  • ● Operational runbook documented
