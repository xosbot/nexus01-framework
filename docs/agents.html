Agents

NEXUS-01 has 4 specialized agents that communicate via the message bus.

🔍

OSINT Agent

Intelligence Gathering

Searches the web, scrapes content, and produces structured intelligence reports. Uses Firecrawl for scraping with DuckDuckGo fallback.

Capabilities

  • Web search via DuckDuckGo / Firecrawl
  • Content scraping with markdown extraction
  • Structured OSINT reports with confidence scoring
  • LLM-synthesized analysis and recommendations

Usage

# From CLI
osint AI agent frameworks 2026

# From Telegram
osint research competitor X

# Via bus
await bus.request(Message(
    sender="orchestrator",
    recipient="osint",
    type="task",
    payload={"query": "cybersecurity trends 2026"}
))

Report Format

{
  "query": "AI agent frameworks",
  "summary": "Agent frameworks converge on ReAct pattern...",
  "key_findings": [
    "Docker sandboxing is now standard",
    "RAG reduces hallucination by 40-60%",
    "Multi-agent outperforms single-agent"
  ],
  "confidence": 0.82,
  "sources": [...],
  "engines_used": ["firecrawl", "duckduckgo"]
}
⚡

Executor Agent

Command Execution

Runs shell commands and file operations inside a Docker sandbox. Gated by Cold Mode and requires HITL approval for destructive actions.

Capabilities

  • Sandboxed command execution (Docker)
  • File read/write with backup
  • Resource limits: CPU, memory, timeout
  • Network disabled, read-only root

Usage

# From CLI (requires approval)
exec ls -la /tmp

# Via bus
await bus.request(Message(
    sender="orchestrator",
    recipient="executor",
    type="task",
    payload={
        "action": "run_command",
        "params": {"cmd": "python3 --version"},
        "permission": "EXECUTE",
        "confidence": 0.9
    }
))
Sandbox: Commands run in python:3.12-slim containers with 0.5 CPU, 256MB RAM, 30s timeout, no network, read-only root.
📊

Analyst Agent

Pattern Analysis

Analyzes data for patterns, anomalies, and insights. Uses RAG for context and LLM for reasoning.

Capabilities

  • Data pattern analysis
  • Anomaly detection
  • RAG-grounded reasoning
  • Structured reports

Usage

# From CLI
analyst analyze this dataset for anomalies

# Via bus
await bus.request(Message(
    sender="orchestrator",
    recipient="analyst",
    type="task",
    payload={"data": {"input": "sales data Q1-Q4"}}
))
🧠

Orchestrator

Reasoning + Delegation

The brain. Routes intent to the right agent, runs the ReAct loop, and chains multi-step tasks.

Capabilities

  • Intent classification and routing
  • ReAct agent loop (reason + act)
  • Multi-agent task chaining
  • Tool delegation (search_knowledge, ask_osint, ask_analyst)
  • RAG context injection

Usage

# Natural language — auto-routed
research AI agents and analyze the risks

# The orchestrator:
# 1. Classifies intent → OSINT + Analyst
# 2. Runs ReAct loop with tool calls
# 3. Chains: ask_osint("AI agents") → ask_analyst(result)
# 4. Returns synthesized response

Adding Custom Agents

Create a new agent by extending BaseAgent:

from agents.base import BaseAgent
from core.bus import Message

class CustomAgent(BaseAgent):
    def __init__(self, llm, memory, rag=None):
        super().__init__("custom", llm, memory, rag)
        self.tools = {
            "my_tool": self._my_tool,
        }

    async def on_message(self, message: Message) -> dict:
        task = message.payload.get("task", "")
        result = await self.act("my_tool", input=task)
        return {"analysis": result}

    async def _my_tool(self, input: str) -> str:
        return f"Processed: {input}"

Then wire it in core/app.py:

custom = CustomAgent(llm, memory, rag)
custom.set_bus(msg_bus)
