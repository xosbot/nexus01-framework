Architecture Patterns Explained

The four dominant approaches to building agentic systems

CrewAI · AutoGen

Role-Based

Agents have roles, goals, and backstories. A manager agent delegates tasks to worker agents based on their specialties. Simple to understand, natural to reason about, easy to extend with new roles.

Crew(researcher, writer, reviewer)
→ manager.assign(task, agent)
LangGraph

State Machine

Nodes are computation steps. Edges define transitions based on conditions. Durable execution means state survives crashes. Best for complex, branching workflows that need persistence.

graph.add_node("fetch", fetch_data)
graph.add_edge("fetch", "analyze")
AutoGen

Conversation

Everything is agent-to-agent dialogue. A GroupChatManager selects which speaker talks next. Agents debate, refine, and converge on answers through structured discussion.

GroupChat(agents=[a1, a2, a3])
manager.select_next_speaker()
SmolAgents

Code-First

Agents write Python code, not JSON. The framework parses and executes agent-generated code directly. 30% fewer steps than tool-calling approaches for many tasks.

@tool
def search(query):
return web_search(query)

What NEXUS-01 Takes From Each

From LangGraph

Communication Protocols: MCP vs A2A

Two protocols, two directions — together they form the complete nervous system

MCP

Model Context Protocol

Agent ↔ Tool communication

  • 97M+ downloads, Linux Foundation governance
  • Primitives: Tools, Resources, Prompts
  • JSON-RPC 2.0 over HTTP/StdIO
  • Standardized tool discovery and invocation
  • Secure, sandboxed execution environment
Used By
Claude Desktop, Cursor, ChatGPT, Windsurf, Zed, VS Code

A2A

Agent-to-Agent Protocol

Agent ↔ Agent communication

  • 150+ organizations in production
  • Agent Cards for discovery and capability negotiation
  • HTTPS + SSE for streaming responses
  • Cross-platform agent interoperability
  • Task lifecycle management with status tracking
Used By
Salesforce, Accenture, MongoDB, SAP, Atlassian, Google
The Big Picture MCP is vertical (agent → tool). A2A is horizontal (agent → agent). Together they form the complete nervous system of an agentic architecture.

When to Use What

A decision guide for choosing the right framework for your project

Choose CrewAI if...

  • → You need business automation fast
  • → Your team knows Python but not AI
  • → You want role-based agent design
  • → You need production-ready features

Choose LangGraph if...

  • → You need complex state management
  • → Your workflow has branching paths
  • → You need durable execution
  • → You want LangSmith observability

Choose NEXUS-01 if...

  • → You want to learn how agents work
  • → You need zero-cost local development
  • → You want full control over every layer
  • → You're building personal tools

Choose OpenAI SDK if...

  • → You want the simplest possible API
  • → You're prototyping, not production
  • → You need agent handoffs
  • → You're already in the OpenAI ecosystem

Choose AutoGen if...

  • → You want agent debate/consensus
  • → Your task benefits from discussion
  • → You're doing research/experimentation
  • → You need multi-agent conversation

Choose Claude SDK if...

  • → You need code generation
  • → You want tool-use loops
  • → You're building coding assistants
  • → You need sub-agent spawning
