The SKILL.md Format

A standardized format for agent instruction files — progressive loading, portable across platforms.

📄 Format Structure

# SKILL.md --- name: "web-scraping" description: "Extract data from URLs" version: "1.0.0" tags: ["osint", "web"] ## Instructions When to activate this skill... 1. Validate URL format 2. Set timeout and headers 3. Parse response with BS4 4. Store in ChromaDB ## Resources - scripts/scraper.py - templates/report.md

⚡ Progressive Loading

1

Metadata

Name, description, tags — loaded first for routing decisions

2

Instructions

Core logic, decision trees, when-to-activate rules

3

Resources

Scripts, templates, reference docs — loaded on execution

The SKILL.md format enables progressive loading: metadata is read first to determine if a skill matches, instructions are loaded only when the skill is activated, and resources are fetched on-demand during execution. This keeps token usage minimal while maintaining full capability.

Karpathy's 4 Principles for AI Coding

The rules that reduce agent errors from 41% to 11% — battle-tested guidelines from one of the most respected voices in AI.

1 THINK BEFORE CODING

The most common failure mode is jumping to code before understanding the problem fully. Agents that pause to think produce dramatically fewer bugs.

Before (41% errors)
Immediately generates code from first token. No planning, no context gathering.
After (11% errors)
Reads codebase, gathers context, plans approach, then writes focused implementation.

2 SIMPLICITY FIRST

Over-engineering is the silent killer of AI-generated code. The best agents choose the simplest solution that works, then iterate only when needed.

Before
Creates abstract factory patterns for a 20-line function. Adds 15 dependencies.
After
Writes a single function. Adds complexity only when requirements demand it.

3 SURGICAL CHANGES

Rewriting entire files when a 3-line edit would suffice. Targeted changes reduce regression risk by an order of magnitude.

Before
Rewrites 500 lines to fix a single off-by-one error. Breaks unrelated features.
After
Edits exactly 3 lines. Tests the specific behavior. Nothing else changes.

4 GOAL-DRIVEN EXECUTION

Lost agents produce code that technically works but misses the actual goal. Clear goal articulation prevents costly detours.

Before
Implements what was literally asked. Misses the business intent. Rework required.
After
Asks clarifying questions. Validates understanding. Builds the right thing first.

Build Your Own Skills

A step-by-step guide to creating custom SKILL.md files for your agents.

Step 1 Define the Skill

Start with a clear problem statement. What does this skill do? When should it activate?

# Define your skill metadata name: "email-triage" description: "Sort and prioritize emails" triggers: - "new email arrived" - "check inbox"

Step 2 Write Instructions

Define the decision tree. Be explicit about conditions, thresholds, and fallbacks.

## Instructions when "email contains attachment": 1. Extract attachment metadata 2. Classify by file type 3. Store in appropriate folder 4. Notify user of receipt when "email is newsletter": 1. Mark as read 2. Archive after 24h

Step 3 Add Resources

Link scripts, templates, and reference files that the skill needs during execution.

## Resources scripts: - scripts/parse_email.py - scripts/classify.py templates: - templates/digest.md references: - docs/priority-rules.md

Step 4 Test & Iterate

Activate the skill, observe agent behavior, refine instructions based on real outcomes.

# Test the skill via Telegram /send "Process my inbox" # Agent will: # 1. Match email-triage skill # 2. Load instructions # 3. Execute scripts # 4. Return results

Browser Harness & Computer Use

Three approaches to giving agents eyes and hands on the web.

🌐

Playwright MCP

Model Context Protocol server for browser automation. Structured, reliable, testable.

  • Headless Chromium automation
  • Screenshot & DOM extraction
  • Form filling & navigation
  • Network interception
  • MCP-native tool interface
🤖

Claude Computer Use

Anthropic's vision-based approach. The model sees screenshots and moves the mouse.

  • Screenshot-based reasoning
  • Pixel-level mouse control
  • Keyboard input simulation
  • Works with any application
  • No DOM access needed
👁

Browser Use Framework

Open-source framework for building browser-using agents with custom workflows.

  • Python SDK for browser control
  • Multi-tab management
  • Cookie & session handling
  • Visual element grounding
  • Custom action pipelines
